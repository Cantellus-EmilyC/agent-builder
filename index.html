Overview
Build Your First AI Agent
This hands-on simulation walks you through the process of building an AI agent from scratch — no technical background required. You will end up with a working agent by the end.
This is not the same as chatting with an AI. Most people have tried asking ChatGPT or Claude a question. Building an agent is a different activity. Here is the key difference.
Chatting with AI
You provide all the context each time
Generic responses, written for anyone
You guide every step of the conversation
Good for one-off questions
Starts from scratch every time
Building an Agent
You design the instructions once, up front
Tailored to a specific job and audience
The agent follows your design on its own
Built for a recurring task or workflow
Consistent, repeatable results
What you will build

A packing list agent — an AI that asks a traveller a few structured questions and returns a complete, personalised packing list based on their trip. Simple enough to build in under 30 minutes. Practical enough to use in real life.

Build — Step 1 of 4
Choose Your Platform
Every AI agent runs on two core components: a language model that understands and generates text, and an orchestration layer that manages the workflow. Both are pre-selected for this simulation.
In a real deployment, you would choose these based on your organisation's needs, existing infrastructure, data privacy requirements, and the technical capabilities of your team.
Selected
🧠
Language Model
Claude by Anthropic
The reasoning engine at the centre of the agent. It reads your instructions, understands what users say, and generates responses. Other options include GPT-4 (OpenAI), Gemini (Google), and Llama (Meta).
The language model is what most people think of as "the AI." It does the thinking — but on its own, it cannot run a repeatable workflow without an orchestration layer.
Selected
⚙️
Orchestration Layer
This Workshop Platform
The system that manages the flow of information — between the user, the language model, and any connected tools. It ensures the agent follows your design from one conversation to the next.
Think of it as the manager who holds the playbook, routes requests to the right people, and keeps the process consistent. The language model executes — the orchestration layer coordinates.
Build — Step 2 of 4
Write Your Agent's Instructions
A system prompt is the set of instructions you give the agent before it ever speaks to a user. It defines the agent's purpose, personality, and rules. The user never sees it — but everything the agent does is guided by it.
Think of it as the employee handbook your agent reads on its first day. It tells the agent why it exists, what it should do, how it should behave, and what it should never do.

Give Your Agent What It Needs to Know
Agents work with two kinds of information. Understanding the difference is one of the most important concepts in agent design.
The information you provide here is loaded into the agent before any conversation begins. It is available to every user, every time, without the agent needing to ask for it.
Built-in information

Provided by you now, before any user interaction. Stays the same across all conversations.

  • Airline carry-on restrictions
  • Seasonal packing tips by climate
  • Your organisation's travel policy
  • A standard packing checklist
Collected in real time

Gathered from the user during the conversation. Unique to each person and each request.

  • Travel destination and dates
  • Airline and bag allowance
  • Planned activities
  • Personal preferences

Overview
Build Your First AI Agent
This hands-on simulation walks you through the process of building an AI agent from scratch — no technical background required. You will end up with a working agent by the end.
This is not the same as chatting with an AI. Most people have tried asking ChatGPT or Claude a question. Building an agent is a different activity. Here is the key difference.
Chatting with AI
You provide all the context each time
Generic responses, written for anyone
You guide every step of the conversation
Good for one-off questions
Starts from scratch every time
Building an Agent
You design the instructions once, up front
Tailored to a specific job and audience
The agent follows your design on its own
Built for a recurring task or workflow
Consistent, repeatable results
What you will build

A packing list agent — an AI that asks a traveller a few structured questions and returns a complete, personalised packing list based on their trip. Simple enough to build in under 30 minutes. Practical enough to use in real life.

Build — Step 1 of 4
Choose Your Platform
Every AI agent runs on two core components: a language model that understands and generates text, and an orchestration layer that manages the workflow. Both are pre-selected for this simulation.
In a real deployment, you would choose these based on your organisation's needs, existing infrastructure, data privacy requirements, and the technical capabilities of your team.
Selected
🧠
Language Model
Claude by Anthropic
The reasoning engine at the centre of the agent. It reads your instructions, understands what users say, and generates responses. Other options include GPT-4 (OpenAI), Gemini (Google), and Llama (Meta).
The language model is what most people think of as "the AI." It does the thinking — but on its own, it cannot run a repeatable workflow without an orchestration layer.
Selected
⚙️
Orchestration Layer
This Workshop Platform
The system that manages the flow of information — between the user, the language model, and any connected tools. It ensures the agent follows your design from one conversation to the next.
Think of it as the manager who holds the playbook, routes requests to the right people, and keeps the process consistent. The language model executes — the orchestration layer coordinates.
Build — Step 2 of 4
Write Your Agent's Instructions
A system prompt is the set of instructions you give the agent before it ever speaks to a user. It defines the agent's purpose, personality, and rules. The user never sees it — but everything the agent does is guided by it.
Think of it as the employee handbook your agent reads on its first day. It tells the agent why it exists, what it should do, how it should behave, and what it should never do.
